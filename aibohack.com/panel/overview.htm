Super PDA

The heart of your display panel is a pretty good Windows CE PDA.

The hardware capabilities (V150):
  • 400MHz XScale (ARM) processor
  • 32MB ROM (over 20MB free)
  • 64MB RAM (over 40MB free when MIRA not running)
  • 1024x768 x 16bit color display
  • Touch screen / Stylus input
  • Two USB ports
  • PCMCIA slot
  • Microphone and speaker
  • Good battery life

The software has much of what is needed:
  • WindowsCE 4.1 core software (not a full PDA version)
  • Viewsonic MiraShell program (the one you are used to)
  • Microsoft provided MIRA shell (with more/different features)
  • Control panel and other standard WindowsCE features
  • some USB drivers, PCMCIA drivers, printer drivers (lots of things underexploited)
But Viewsonic thinks of their Smart Display products as a closed system, where you can't add you own software.

The normal use of a Smart Display is as a "Remote Desktop Client"
The ViewSonic developers have created their own MIRA client with setup options and other things. This is what you are familar with if you've used a V110 or V150.

Alternative Client

You normally can only have one of the ViewSonic's MiraShell programs running at a time (that's all you see). With the Super-PDA hack, you can decide what programs to run, and when.

I prefer to use the Microsoft Terminal Services client (mstsc.exe). It has a few additional customization features not normally available (for example):

You can "Show contents of windows while dragging" and it works pretty well.
If you have installed the Microsoft Remote Desktop Client on another computer (Eg: Windows 95, 98, ME NT, 2K, XP, and even the Mac), you have an idea of what the Microsoft client looks like.

Different clients/Multiple clients (on the same panel)

With the Super-PDA hack, you can decide what programs to run, and when. You can run multiple clients at the same time (attached to different computers). You just need to switch (using the WinCE taskbar) between different clients to switch between desktop computers. With the normal Viewsonic shell, you need to disconnect from one, then reconnect to the other.

NOTE: This is not earth-shaking. You can easily have multiple client sessions running on a laptop or other computer that runs the Remote Desktop Client program (Win95 and up, WinCE, Mac or Linux). Doing it on the closed system Viewsonic panel is new and cool.

NOTE: there are several ways of performing multi-tasking. The easiest is to have the WinCE taskbar "auto-hide" and the Windows XP-Pro taskbar normal. When running programs on the desktop (Windows XP-Pro), you have the entire screen except for the very bottom of the panel. To pop up the WinCE taskbar, you just click the very bottom of the screen. For the screen shot below, I expanded both taskbars for illustration (you can auto-hide one or both).

Multi-Tasking : Many things in one Panel

There are many things going on here:
  • There are two separate MIRA clients running on the Panel (one is shown, the other is minimized)
  • The ViewSonic MIRA shell (minimized) coexists with the Microsoft shell (maximized)
  • When the Microsoft shell/client is running, it has a cute little title bar on the top of the screen ("bmgm24"). You can turn this off.
  • There are two taskbars in this case. The bottom one is the WinCE taskbar (switching between apps on the panel). The other one is from the MIRA session (and is the taskbar of Windows XP-Pro running on the connected desktop)
  • The blue background, Command Prompt and Explorer tree view are running on the desktop computer (ie. any old Windows XP-Pro program)
  • The "Mini Registry Viewer" program is a WinCE program running directly on the Panel (and temporarily on top of the MIRA client)

    Day2.5: File Sharing

    In the file view of the above screenshot. The WindowsCE file system (on the Panel) is exported to the desktop (via WiFi) as a remote network drive, and is displayed on the WindowsXP-Pro desktop as a normal folder. The graphics rendition of that folder is transmitted to the panel and show graphically using the MIRA client.
    So you can use Windows XP tools (eg: compilers, editors) to modify WindowsCE programs/files - without touching the keyboard or mouse of your Windows XP computer.
