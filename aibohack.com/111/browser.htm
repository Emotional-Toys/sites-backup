ABrowser - the AIBO Browser for ERS-110 and ERS-111

Platforms

The AIBO Browser (ABrowser.exe) is a Win32 program. It should work on Windows 95 and above. It should work with both ERS-110 and ERS-111 data. The 3D Virtual AIBO needs DirectX 5 or better (used to playback motions).
The file VIRTAIBO.DLL must be in the same directory as the main program.

Drives

The browser will look for the first OpenR installation it finds by scanning the root drives from 'A:' to 'Z:'. Otherwise it will prompt you for a directory to open.
The browser needs most of the AIBO files to function, so I recommend running it on the actual memory stick, or a image of the memory stick. Version .9 and above do not change any data on the memory stick.

User Interface


The first window pane has a tree of possible files (or other features). Select this first.
The second window pane has a list of nodes which change depending on the file selected. This can contain motion or sound names, behavior state numbers, action names or nothing at all.
The third window pane is the information pane that gives information about what is selected in the previous two window panes.
Poke around and explore. The information pane on the right hand side give you a small amount of context sensitive help.

Summary Mode

This gives a high level dump of the state of you AIBO. This includes the "Quality Time Spent" in the current mode, and how far your dog has evolved at this stage, and when it should evolve to the next stage.

Instinctive Behavior Browsing

Click here for more information on the behavior model.
Select the behavior file (in the first window pane). Behavior files are categorized by the stage of AIBO (eg: "child stage").
After selecting a behavior file, select a state in the middle window pane. The behavior for that state will be shown in the information pane. The format is a BASIC-like pseudo code showing the behavior rules.
Cool features: You can select a behavior rule in the information pane. The "Play" button will play the sound associated with the selected behavior rule. The "Browse Action" button will spawn a new browser giving you the information about the action associated with selected rule. These buttons will be disabled if the don't apply to the currently selected behavior rule. Double clicking on a rule will move you to the next state.

State Descriptions

When displaying state names, the browser will look for a file containing comments (COMMENTS.INI in the same directory as ABROWSER.EXE).
I've included the state names/descriptions I've made up for some of the behavior files, mostly for Adult#1. You can customize this yourself (just edit the COMMENTS.INI file). The format is pretty simple, just read the file.
If you decode more, please send me your improved COMMENTS.INI file so I can include it in future releases.

Learned Behavior Browsing

This is an extension to the Instinctive Behavior mode, but only shows those behavior states and rules where your AIBO is different from the default "instinctive" behavior.
You will notice the information dump is slightly different in that it contains ywo percentages. The first percentage is the instinctive probability, the second percentage is how your dog's behavior.

Action Browsing

This shows all the possible actions of the AIBO. It gives you information about where the action sound and motion data is stored. The "Play Sound" button (when enabled) will play the sound and motion part of the action.
Click here for more information on Virtual AIBO.

Data Files Browsing

This mode lets you browse the "ODA" data files that contain all the motions, sounds and led performances of the AIBO. The "Play" button will play the data. Sounds (WAVs and MIDs) are played as sounds. Motions are played using Virtual AIBO. LED files are played as MIDI files.

System Files Dump

Low level dump of some of the system files.
