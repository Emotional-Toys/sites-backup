[AiboHack] PleoRB Hardware Overview

PleoRB Hardware Overview

Please refer to my original UGOBE Pleo teardown for background. It is old, but still accurate.
For a general overview of the older models, I recommend the excellent How Stuff Works article by Tracy Wilson.


Much of the PleoRB is similar to the original model. This page focuses on changes in the PleoRB hardware.

Overview of Hardware Changes

External Hardware

  • New battery (old NiMHd 2.2AHr, new lighter LiPoly 2.8AHr). Charger logic much improved.
  • USB connection moved inside of battery compartment. Looks like old terminal feature not supported.
  • No hardware serial connector. Much harder for debugging remote control apps.
  • SD card replaced with micro SD card inside of battery compartment. Makes swapping personalities more difficult.
  • Real-Time-Clock battery in foot.

Internal Hardware - Main Body

  • DataFlash in the main body doubled in size (4MB -> 8MB). Default personality program (PLEOPM) needs it.
  • Main CPU appears to be same (Atmel AT91SAM7S256). Still only 64KB of RAM (KiloBytes is not a typo).
  • 3D axis accelerometer in upper body (KXTF9 on I2C bus along with another chip)
  • Additional touch sensors (now 2 on back, 2 on butt/arse)
  • Much of the rest looks similar to original Pleo (Toshiba TMP85FH47 motor controllers, etc).

Internal Hardware - Head

  • Overworked head CPU replaced with two chips (new chip specs unknown).

      Before
    • The old Pleo head CPU did poor sound direction detection, and slow camera capture.
    • The old chip was ARM based with 32KB of flash ROM for code. The old chip did everything: sound detection, camera interface and various head specific features.
      After (PleoRB)
    • The new PleoRB does true voice commands (8 of them), faster camera access (?) and other things.
    • These tasks are split between two different CPUs:
    • The smaller chip has 128KB of flash ROM for code. Chip specs unknown. Probably doing the voice recognition feature. Appears similar to the RSC-300 feature of the iCybie. Lets you record 8 unique speaker dependent voice commands and use them to trigger specific modes.
    • The larger new chip has 1MB of flash ROM. Chip specs unknown. Probably doing the camera interfacing and other head sensor control.
    • Neither of these chips are meant to be user programmable. The Pawn personality program (interpreted by the main CPU in the body) is programmable with the PDK.

  • RFID detection in mouth. See this page for the list of known RFID toys.
  • Additional touch sensors (2 new ones in cheeks)
  • Additional sensors (light, temperature)
  • Head servo configuration changed
  • Still only 64KB in serial SRAM (used for image capture)
  • Camera marked "JS668850" - same as last version?
  • Slow data connection between head and body is claimed to be fixed - but needs testing with real timing.

Software/Firmware

There are several different pieces of firmware/software in the Pleo.
  • The main personality. Often called "PLEOPM". Closed source. A "Pawn" interpreted program. Program and data (sound, motions etc) filling most of the 8MB of data store. See this page for more info.
  • The main CPU firmware. Closed source. ARM code filling most of the 256KB of Flash ROM in the main CPU. Probably very similar to the old version.
  • The two head CPUs have their own firmware. I assume they have a similar update mechanism like the original model. Much more complicated firmware for camera and sound features.
  • The motion controller firmware. Closed source. Toshiba TMP (Z80-like) code in the 4 motion/servo controllers. Probably very similar to the old version.
Note: AFAIK all software/firmware is updatable (typically by installing a special update program in the SD card).
Note: all software/firmware is currently closed source and proprietary.
