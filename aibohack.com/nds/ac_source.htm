Animal Crossing Map Editor (PC Windows App) source and notes

Source code - Warning

For most people, source code is not as useful as you think it will be!
The map editor changes the gamesave (ROMSAVE) that contains your town. The Action Replay MAX/DUO uploads those changes to the AC:WW game cartridge.

The map editor is a convenient user interface for tweeking the gamesave. It doesn't let you do anything that isn't already in the AC:WW game.

So if you think you can create a new feature, like placing full furnature items outside of your house, you will be mistaken. The hard work is figuring out what bytes can be changed in the gamesave, and to figure out what strange things you can trick the game into doing. For example girl faces on boy characters, or funny snowmen sizes.

The actual map editor is a Visual C++ project. It can be compiled with the (not free) Visual C++ 6.0 IDE. It can also be compiled with (the free demo version of) Visual Studio 2005. Optionally you can compile it with the free version of Visual C++ Express 2005 IDE (after the demo version expires). If you don't have theses tool or something compatible, you won't be able to build the project.
The C++ source code is minimally documented. It has features slapped on (based on almost a year of small changes), so it isn't the cleanest code. If you are looking to create something totally different, you may be better off starting with a tool you are more familiar with (eg: Visual Basic).

Download the source code to the current version here AnimalMap 1.8 source (with VC6 and VStudio 2005 projects). Please read the README.TXT files.

ROMSAVE.TXT, technical description of gamesave

Here's a text file listing some of the more important byte offset locations of the gamesave (ROMSAVE). This may be more interesting to you if you are trying to find new tricks/features.
This info can also be used to make standalone "AR DS" codes (the gamesave/ROMSAVE is stored in RAM in the exact same format, so you only have to add a versions specific offset)
This is also a very technical document, talking about hex bytes and offsets and minimally understood values. For adding new features, you will need to do some experimentation and guessing.
